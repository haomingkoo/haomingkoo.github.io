I Built an AI Wine Deal Hunter — Here's What 50 Bottles Taught Me

How I built an automated wine price comparison tool for Singapore that compares Platinum Wine Club vs Grand Cru vs Vivino, using web scraping, Gemini AI with search grounding, and a self-sustaining daily pipeline.

The problem — am I getting ripped off?

It started with a simple question: am I getting ripped off on wine?

I'm a member of Platinum Wine Club in Singapore. They send monthly selections — nice Burgundies, Barolos, the occasional Champagne. The prices seem reasonable, but "seems reasonable" isn't a number. I wanted to know: how do these prices actually compare to other retailers? Am I getting member value or member theatre?

The obvious move is to check each wine on Grand Cru (another big Singapore retailer) and Vivino (the global wine marketplace). Doing this manually for one wine takes about five minutes. For 50 wines, that's over four hours of tab-switching and squinting at price labels. So I did what any reasonable engineer would do and automated the whole thing.

🇬🇸
Singapore only. This tool compares Singapore wine retailers (Platinum Wine Club and Grand Cru). Prices are in SGD. Vivino prices reflect the Singapore market where available.

Try it yourself

Head to wine.kooexperience.com. You'll see a table of ~50 wines with prices from Platinum, Grand Cru, and Vivino side by side. Each wine gets a deal score from 0 to 100, colour-coded so the best deals pop out immediately.

You can sort by deal score, filter by wine type (red, white, sparkling), or click into any wine to see a breakdown of why it scored the way it did. There's also a map showing where each wine comes from, because I'm a sucker for Leaflet maps and any excuse to add one.

The data refreshes daily via a cron pipeline. No manual intervention. I built it once and it just runs, like a Roomba for wine deals.

The numbers

After tracking 50 wines across three sources, here's what the data says:

  • 16 out of 50 wines (32%) are genuinely cheaper on Platinum than Grand Cru.
  • Average Vivino rating: 4.09 out of 5 — the selection quality is solid.
  • Price range: $45 to $780 per bottle.
  • Best deal: Gevrey-Chambertin by Domaine Claude Dugat — $170 on Platinum vs $1,020 on Grand Cru. That's 83% cheaper, though it's a bundle of 3 bottles, which is doing a lot of heavy lifting in that comparison.
  • Worst "deal": Gattinara Molsino — 25% MORE expensive on Platinum than Grand Cru. Ouch.
  • Most reviewed: Miraval Rosé 2024 with 58,336 Vivino ratings. More on this legend later.

The short version: Platinum isn't a scam, but it's not universally cheaper either. About a third of the wines are genuine deals. The rest are priced similarly or sometimes higher. Knowing which is which — that's the whole point of this tool.

How the pipeline works

The system runs as a daily cron job on Railway. Here's the flow: 

┌────────────────────────────────────────────────────────────┐
│  Daily Pipeline (cron @ 06:00 SGT)                         │
│                                                            │
│  1. SCRAPE                                                 │
│  ├── Selenium → Platinum Wine Club (prices + labels)       │
│  └── Selenium → Grand Cru (matched wines + prices)         │
│                                                            │
│  2. MATCH                                                  │
│  ├── Fuzzy string matching (wine names across retailers)   │
│  └── Manual overrides for tricky labels                    │
│                                                            │
│  3. ENRICH (Vivino)                                        │
│  ├── Brave Search API → find Vivino URLs                   │
│  ├── Gemini AI (search grounding) → extract:               │
│  │   ├── Vivino rating + review count                      │
│  │   ├── Average price (SGD)                               │
│  │   └── Wine metadata (region, grape, style)              │
│  └── Fallback: manual CSV for wines Gemini can't resolve   │
│                                                            │
│  4. SCORE                                                  │
│  └── Compute deal score (0-100) per wine                   │
│                                                            │
│  5. SERVE                                                  │
│  ├── FastAPI → /api/wines (JSON)                           │
│  ├── PostgreSQL (Railway) for persistence                  │
│  └── Static frontend (vanilla JS + Leaflet.js)             │
└────────────────────────────────────────────────────────────┘

The scraping layer uses Selenium with headless Chrome. Both Platinum and Grand Cru render their catalogues with JavaScript, so a simple HTTP request won't cut it. Selenium waits for the page to fully render, then extracts wine names, vintages, regions, and prices from the DOM.

The matching step is trickier than it sounds. "Chambolle-Musigny 1er Cru Les Amoureuses 2020" on one site might be listed as "Domaine Serveau Chambolle Musigny Amoureuses Premier Cru 2020" on another. Fuzzy matching with a similarity threshold of 0.75 catches most of these, but some wines need manual overrides stored in a CSV.

The Vivino problem (and the AI fix)

This was the part that almost killed the project. I needed Vivino data — ratings, review counts, average prices — to make the deal score meaningful. The problem? Vivino blocks everything.

I tried urllib. Blocked. I tried requests with custom headers. Blocked. I tried Playwright with stealth mode. Blocked. I tried rotating proxies. Still blocked. Vivino's anti-bot detection is genuinely impressive (and genuinely annoying). Every server IP — Railway, AWS, GCP, you name it — gets detected and rejected.

The breakthrough came from a weird direction: Gemini AI with Google Search grounding.

Google's Gemini API has a feature called search grounding where you can ask Gemini a question and it will search the web in real time to find the answer. Crucially, Google's own crawlers can access Vivino just fine. So instead of scraping Vivino directly, I ask Gemini:

"What is the Vivino rating, number of reviews, and average Singapore price for Domaine Claude Dugat Gevrey-Chambertin 2021?"

Gemini searches, finds the Vivino page, and returns structured data. It's not scraping — it's having an AI assistant do the research for me. The success rate is about 82% automated. For the remaining 18%, the wine is either too obscure for Vivino or Gemini returns ambiguous results. Those get flagged for manual review.

Before Gemini, I also use the Brave Search API to find the exact Vivino URL for each wine. This gives Gemini a direct link to work with rather than relying on its own search, which improves accuracy significantly.

The deal score

Every wine gets a score from 0 to 100. The formula weights four components:

  • Price gap (40 points): How much cheaper is Platinum compared to Grand Cru and Vivino's average price? Bigger savings = more points. If Platinum is more expensive, this goes negative and drags the score down.
  • Vivino rating (25 points): Higher-rated wines score higher. A 4.5-rated wine at a discount is more interesting than a 3.5-rated wine at the same discount. Quality matters.
  • Rating confidence (20 points): A 4.2 rating backed by 10,000 reviews is more reliable than a 4.2 with 12 reviews. This component scales with review count, rewarding wines with strong community consensus.
  • Market position (15 points): Where does the Platinum price sit relative to the overall market? If it's in the bottom quartile of known prices, that's a strong signal.

The result is a single number that answers: "Is this wine a good deal on Platinum?" A score above 70 means you're probably getting genuine value. Below 40 means you should price-check before buying. Scores are colour-coded on the frontend — green for great deals, red for overpriced ones.

Surprising finds from the data

After staring at 50 wines for longer than is healthy, a few patterns jumped out:

Rosé has the best value. The Miraval Rosé 2024 — yes, the one from Brad Pitt and Angelina Jolie's Provence estate, a.k.a. the world's most famous divorce wine — is consistently cheaper on Platinum than anywhere else. With 58,336 Vivino reviews and a solid rating, it's the highest-confidence deal in the dataset.

Champagne is mostly the same price everywhere. The big Champagne houses (Moët, Veuve Clicquot, etc.) have tight distribution agreements. Price differences across retailers are usually under $5. Not much to optimise here.

Italian wines need checking. The biggest price variances showed up in Italian bottles — Barolo, Barbaresco, Gattinara. Some were great deals on Platinum; others were significantly overpriced. Italian wine pricing in Singapore appears to be all over the place.

Burgundy is where the real deals hide. The Gevrey-Chambertin by Domaine Claude Dugat at 83% cheaper is the extreme case, but several other Burgundies showed 20-40% savings. These tend to be smaller producers where Platinum's direct import relationships actually matter.

The stack

Backend:    Python 3.12 + FastAPI + Uvicorn
Scraping:   Selenium (headless Chrome)
Search:     Brave Search API (Vivino URL discovery)
AI:         Gemini 2.0 Flash (search grounding for Vivino data)
Database:   PostgreSQL (Railway managed)
Frontend:   Vanilla JS + Leaflet.js (wine origin map)
Hosting:    Railway (backend) + static frontend
Scheduling: Railway cron (daily 06:00 SGT)
Cost:       ~$0/month (all free tiers)

The entire thing runs on free tiers. Railway's free plan covers the backend. Brave Search has a generous free tier. Gemini's free API handles the volume easily (50 wines is nothing). PostgreSQL is included with Railway. The only thing I pay for is my time, which I choose not to quantify because it would be depressing.

Lessons learned

Don't fight the wall, go around it. I spent days trying to scrape Vivino directly — urllib, Playwright, stealth plugins, rotating headers. None of it worked. The moment I stopped trying to beat their bot detection and instead asked an AI to do the lookup through Google, it just worked. Sometimes the clever solution is admitting the brute-force approach won't win.

Free tiers go surprisingly far. This whole project runs on $0/month. Railway, Gemini, Brave — all free tiers. For a side project serving one person's wine purchases, you don't need infrastructure that scales to millions. You need infrastructure that works reliably for 50 wines a day.

Data quality beats data quantity. Early on I was focused on getting more wines, more sources, more data points. But the real value came from getting accurate prices and verified Vivino matches for the wines I already had. A wrong price is worse than no price.

Build for yourself first. This tool exists because I genuinely use it before every wine purchase. That's the best feature spec: your own frustration. If it solves your problem, it'll probably solve someone else's too.

What's next

A few things I want to add:

  • More retailers: Wine Connection, 1855 The Bottle Shop, and other Singapore wine stores. More data points = better price intelligence.
  • Price history: Track prices over time to spot trends. Does Platinum get cheaper before Christmas? Do Grand Cru prices spike during promotions?
  • Alert system: Get notified when a high-rated wine drops below a price threshold. Because checking a website daily is exactly the kind of manual work this project was supposed to eliminate.
  • Better Vivino coverage: Push that 82% automation rate higher. Some wines just need better search queries or alternative data sources.
